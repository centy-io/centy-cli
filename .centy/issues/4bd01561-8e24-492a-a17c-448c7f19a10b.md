# Umbrella: Consolidate all CRUD commands to generic `centy <operation> <type>` format

All CRUD CLI commands should use a unified `centy <operation> <type>` format that works with any registered item type — built-in (issues, docs) or custom (epics, bugs, tasks, etc.). The daemon already has generic gRPC endpoints for all operations; the CLI needs to be updated to use them.

## Target CLI format

```bash
centy create <type> --title "..." [--status ...] [--priority ...]
centy get <type> <id> [--json] [--global]
centy list <type> [--status ...] [--priority ...] [--json]
centy update <type> <id> [--title ...] [--status ...] [--priority ...]
centy delete <type> <id> [--force]
```

Where `<type>` is any registered item type: `issue`, `doc`, `epic`, `bug`, `task`, etc.

## Daemon generic endpoints (all already implemented)

- `CreateItem` — `grpc-client-items.ts` / `item_create.rs`
- `GetItem` — `grpc-client-items.ts` / `item_get.rs` (or equivalent)
- `ListItems` — `grpc-client-items.ts` / `item_list.rs` (or equivalent)
- `UpdateItem` — `grpc-client-items.ts` / `item_update.rs` (or equivalent)
- `DeleteItem` — `grpc-client-items.ts` / `item_delete.rs` (or equivalent)
- `SoftDeleteItem` / `RestoreItem` — also available

## Design considerations

- **Backward compatibility**: `centy create issue` should continue to work. Old commands can become thin aliases or oclif aliases pointing to the generic command.
- **Output formatting**: The generic commands should adapt output based on item type config (display numbers, statuses, priorities, etc.)
- **Type resolution**: The command receives a singular type name (e.g., "issue") and resolves to the plural folder name (e.g., "issues"). The daemon can assist with this via the item type config.
- **Shared flags**: Common flags like `--project`, `--json` should be consistent across all operations.
- **Type-specific features**: Some flags (e.g., `--draft` for issues, `--slug` for docs) are type-specific. These can be handled via `--custom-field key=value` or kept as special cases for built-in types.
