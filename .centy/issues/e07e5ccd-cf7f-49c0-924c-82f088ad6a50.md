# Add ! negation prefix support to --status filter

## Overview

The `--status` flag currently only supports filtering items that _match_ a given status. We need to support excluding items with a specific status using the `!` prefix.

## Use case

```sh
# List all issues that are NOT closed
centy list issues --status '!closed'

# Get next item that is not closed
centy next issue --status '!closed'
```

## Implementation

- Parse status values starting with `!` as a negation filter
- Use `$ne` (not equal) operator in the filter object instead of `$eq`
- Apply to all commands that support `--status`: `list`, `list items`, and `next`
