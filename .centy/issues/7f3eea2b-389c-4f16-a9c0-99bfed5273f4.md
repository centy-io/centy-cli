# Add --json flag to create, update, and delete commands

## Summary

The `create`, `update`, and `delete` commands are missing a `--json` flag for machine-readable output. All other commands (`get`, `list`, `close`, `daemon status`, `version`) already support `--json`. This inconsistency makes it harder to script and automate workflows.

## Commands affected

- `centy create <type>` — no `--json` flag; outputs human-readable string
- `centy update <type> <id>` — no `--json` flag; outputs human-readable string
- `centy delete <type> <id>` — no `--json` flag; outputs human-readable string

## Expected behavior

```bash
# create with JSON output
centy create issue --title "Bug" --json
# => { "id": "...", "displayNumber": 42, "title": "Bug", "status": "open", "type": "issue" }

# update with JSON output
centy update issue 1 --status closed --json
# => { "id": "...", "displayNumber": 1, "title": "...", "status": "closed", "type": "issue" }

# delete with JSON output
centy delete issue 1 --force --json
# => { "id": "...", "type": "issue", "deleted": true }
```

## Implementation

Add a `--json` flag to `src/commands/create.ts`, `src/commands/update.ts`, and `src/commands/delete.ts`. When `--json` is set, output a JSON object instead of the current human-readable string.

## Related

- Consistent with `get`, `list`, `close`, `version`, `daemon status` which all support `--json`
