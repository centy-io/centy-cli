# Require git repository for centy init with opt-out flag

When creating a centy folder via centy init:

1. Check if the current directory is inside a git repository
1. If not inside a git repo, block initialization unless a flag is provided (e.g., –no-git or –skip-git-check)
1. Show a warning/hint that initializing outside a git repository is not recommended

This ensures users don’t accidentally create .centy folders in locations where git-based features won’t work properly.

## Acceptance Criteria

- [ ] Add git repository detection to init command
- [ ] Add –no-git flag to bypass the check
- [ ] Display clear warning when –no-git is used
- [ ] Block init without flag when not in a git repo
