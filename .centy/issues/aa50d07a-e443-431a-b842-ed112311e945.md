# Cascade untrack projects when untracking organization

When untracking an org via CLI and the `ORG_HAS_PROJECTS` error is returned, prompt the user asking if they want to cascade-untrack all projects belonging to the org.

## Current behavior

The CLI shows the error and exits: "Organization has N projects. Reassign or remove them first."

## Desired behavior

When `ORG_HAS_PROJECTS` error is received, prompt:

- "This organization has N projects. Untrack them as well? (y/N)"
- On yes: retry the request with `cascade: true`
- On no: exit with the original guidance

## Technical context

The daemon already supports cascade deletion via the `cascade` flag on `DeleteOrganizationRequest`. This is a CLI prompt + retry change.

## Files to reference

- `centy-cli/src/commands/untrack/org.ts` (untrack org command)
- `centy-cli/src/daemon/` (daemon client for deleteOrganization)
