# Update daemon discovery to support cargo install path

## Summary

With the switch to `cargo install centy-daemon` (centy-daemon#355), the daemon lands in `~/.cargo/bin/` instead of `~/.centy/bin/`. Update the CLI's binary search logic and install-daemon flow accordingly.

## Changes needed

- In `get-search-paths.ts`: add `~/.cargo/bin/centy-daemon` as an explicit search path (before the PATH fallback)
- In `get-install-dir.ts` / `install-daemon`: remove or repurpose the install flow — the CLI no longer needs to install the daemon itself
- Remove `binary-installation.integration.spec.ts` or update it for the new flow
- Update any user-facing messages that reference `~/.centy/bin/`
