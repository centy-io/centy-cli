# Support `centy daemon status` subcommand

Currently, daemon lifecycle commands are top-level: `centy start`, `centy shutdown`, `centy restart`, `centy info`. The only subcommand under `centy daemon` is `autostart`. Add a `centy daemon status` command that reports whether the daemon is running or stopped, daemon version and basic info (when running), and connection details. Reuse `checkDaemonConnection()` and `daemonGetDaemonInfo()`. Support `--json` flag for machine-readable output.
