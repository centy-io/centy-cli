# Consolidate CLI list commands to use generic ListItems RPC

Replace the type-specific `list issues`, `list docs`, `list assets`, `list orgs`, `list users` commands with a single generic `centy list <type>` command that calls the daemon's `listItems` gRPC endpoint.

## Current state

- `src/commands/list/issues.ts` calls `daemonListIssues` (type-specific)
- `src/commands/list/docs.ts` calls `daemonListDocs` (type-specific)
- `src/commands/list/assets.ts`, `list/orgs.ts`, `list/users.ts` — each with their own daemon wrappers
- The daemon already has a generic `ListItems` RPC in `grpc-client-items.ts` that accepts `itemType`, `status`, `priority`, `includeDeleted`, `limit`, `offset`

## Target UX

```bash
centy list issues
centy list issues --status open --priority 1
centy list docs
centy list epics --status in-progress
centy list bugs --json --limit 10
```

Note: The `<type>` argument uses the **plural** form (matching existing convention: `list issues`, not `list issue`).

## Implementation

### New files

- `src/commands/list/items.ts` — Generic oclif command with `<type>` as positional arg. Flags: `--status`, `--priority`, `--json`, `--draft`, `--limit`, `--offset`, `--include-deleted`, `--project`

### Modified files

- `src/commands/list/issues.ts` — Replace with alias/wrapper
- `src/commands/list/docs.ts` — Same
- `src/commands/list/assets.ts` — Same
- `src/commands/list/orgs.ts` — Same
- `src/commands/list/users.ts` — Same

### Notes

- Output formatting should adapt based on item type config — show display numbers only if `displayNumber` feature is enabled, show priority only if `priority` feature is enabled, etc.
- `list projects` and `list link-types` are different entities and may stay separate
- The `--draft` flag is issue-specific; for the generic command, consider whether draft is a universal concept or a custom field
